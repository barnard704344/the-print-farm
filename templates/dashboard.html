Send G-code from OrcaSlicer to the Print Farm

Virtual Printer Setup (Recommended)

Add the farm as a network printer in OrcaSlicer. Slice and send directly — no scripts needed.
Works with any printer profile (BambuLab, Klipper, Voron, etc.)

1
Select Your Printer Profile
In the left sidebar, click the settings icon () on the far right of the "Printer" row to choose or add your printer type (e.g. Bambu Lab P1S, Voron, etc.)
2
Open Connection Settings
Click the wifi icon () next to the printer name to open the connection dialog.
If you don't see the wifi icon, see the BambuLab note below.
3
Set Host Type to Octo/Klipper
Change the Host Type dropdown to Octo/Klipper
This works for all printers including BambuLab — it's just the protocol OrcaSlicer uses to talk to the farm.
4
Enter Hostname & API Key
Copy the Hostname and API Key from the table below for the printer you want to target.
5
Test & Print
Click Test to verify the connection. Then slice as normal and click the upload/print icon to send the G-code to the farm.
⚠ BambuLab printers — extra step

BambuLab printer profiles hide the wifi icon by default. You need to enable it first:

  1. Click the three dots (⋯) next to the printer name and tick "Use 3rd-party print host"
  2. The wifi icon () will now appear — click it to open the connection dialog
  3. Continue from step 3 above

Virtual Printer — LAN Mode (AMS Sync)

Add these directly in OrcaSlicer as a BambuLab LAN printer to get live AMS slot sync. Files you send from Orca will be queued in the farm for staff to assign.

Printer Virtual IP Serial Number Access Code

In OrcaSlicer: Printer → Add → BambuLab → LAN mode — enter the IP, access code and serial above.

Notes

  • Jobs uploaded via a per-printer port are assigned to that printer but not auto-sent — click "Send" in the Job Queue when ready.
  • Jobs uploaded to the general queue (port 80) are unassigned — send to any printer.
  • The farm automatically wraps G-code in .3mf for BambuLab printers.
  • You can also drag & drop files directly into the Job Queue tab.

Active Directory Configuration

Configure LDAP/Active Directory authentication. When enabled, users log in with their AD credentials. Students get print access; staff get full admin access.

Role Mapping (OU Paths)

Users whose DN contains the Staff OU path get full admin access. Users whose DN contains the Student OU path get print-only access.

Obico AI Failure Detection

Connect a self-hosted Obico server to enable AI-based print failure detection for Klipper printers. Configure per printer — each needs its own Obico printer ID.

Spoolman Filament Tracking

Connect to a Spoolman instance to track filament spool inventory and automatically deduct usage when prints complete. The API is available at /api/v1/spoolman/ once configured.
